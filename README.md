## 📖 專案簡介
本專案是為現代化企業設備運維場景量身打造的數位化管理平台。針對傳統工業設備運維中紙質記錄繁瑣、線下排程低效、設備狀態不透明、故障突發率高等痛點，依托主流前後端分離技術棧，實現設備台賬數位化管理、檢修工作流自動化、設備故障智能預警、操作日誌全程溯源等核心能力。

系統摒棄了傳統人工運維模式，通過標準化、流程化、智能化的管理方式，有效提升企業設備利用率，降低設備突發故障風險，實現設備運維全流程精準化、規範化管控。

---

## 🎯 需求分析

### 1. 業務痛點
* **狀態不透明**：設備狀態資訊不透明，無法即時掌握設備運行、檢修狀態。
* **排程效率低**：人工排程效率低，檢修任務易遺漏，存在檢修逾期風險。
* **留存追溯差**：傳統紙質記錄留存困難，人工審計工作量大、追溯性差。

### 2. 核心功能目標
* **精細化權限體系**：基於 RBAC 權限模型，搭建用戶、角色、權限三級細分管控體系，實現細粒度權限控制。
* **全業務閉環管理**：覆蓋設備入庫、台賬維護、檢修計劃生成、檢修執行記錄、故障預警處理全流程，形成完整設備數據業務閉環。
* **全流程安全審計**：基於 AOP 切面編程技術，自動記錄系統核心業務操作日誌，實現所有運維操作可查詢、可追溯。

### 3. 非功能需求
* **數據安全**：用戶密碼採用 BCrypt 單向加密存儲，杜絕明文洩露風險。
* **接口安全**：基於 JWT 實現無狀態身份認證，保障接口訪問合法性。
* **系統性能**：保障接口高可用、低延遲，適配企業日常高頻運維操作場景。

## 🛠️ 專案設計

### 1. 整體設計思路
本專案採用前後端分離架構，以高安全性、高可維護性、低耦合度為核心設計原則，搭建標準化設備運維管理平台。
* **技術架構**：後端基於 Spring Boot 3 搭建 RESTful 標準化 API 接口，前端通過 Vue3 + Element Plus 構建輕量化、高交互的用戶界面，前後端通過接口數據交互，完全解耦。
* **權限設計**：基於 RBAC 角色權限模型，通過數據庫關聯用戶、角色、權限，實現前端路由與後端 API 接口雙重細粒度權限管控。
* **安全防禦**：集成 Spring Security + JWT 實現無狀態登錄認證；BCrypt 加密存儲用戶密碼；通過 AOP 切面編程實現全局操作日誌自動採集與審計，保障操作全程可追溯。
* **開發策略**：採用模組化拆分開發，將系統業務拆解為獨立功能模組，降低代碼耦合度，提升系統可拓展性與維護性。

### 2. 核心模組功能介紹

#### 🔐 用戶與權限模組
系統基礎核心防線，支持用戶登錄、注銷、密碼修改、角色分配、權限指令控制，通過精細化權限配置，區分不同運維人員的操作權限，保障系統訪問安全。

#### 📋 設備資訊管理模組
系統數據基礎模組，實現設備台賬的新增、查詢、修改、刪除全功能管理，支持通過設備名稱、運行狀態等條件精準檢索，完整留存設備基礎檔案資訊。

#### 🔧 檢修管理模組
核心業務模組，分為檢修計劃排程、檢修結果錄入兩大子功能，支持運維人員制定定期檢修計劃、記錄檢修執行結果，實現設備檢修從規劃到落地的全流程閉環管理，規避檢修逾期問題。

#### ⚠️ 預警資訊模組
設備安全保障核心模組，支持動態配置故障預警規則，即時監測設備異常狀態，對觸發的預警資訊提供確認、處理、忽略等操作能力，及時排查設備故障隱患，降低突發故障停機風險。

#### 📊 系統審計模組
依托 AOP 切面技術自動採集系統操作數據，精準記錄操作人、IP 地址、操作時間、請求耗時、操作內容等資訊，自動生成運維操作日誌，滿足企業運維審計與問題追溯需求。

## 📁 系統架構分層
* **頂層**：設備檢修與預警系統（整體業務平台）
* **核心安全層**：JWT 身份認證、權限攔截器、AOP 日誌記錄
* **數據持久層**：MySQL + MyBatis-Plus 數據持久化
* **業務功能層**：設備台賬系統、檢修工作流、預警監控中心、系統管理

## ✅ 專案亮點
* 前後端分離架構，代碼解耦，便於迭代維護與二次開發。
* RBAC 精細化權限管控，適配多角色、多崗位運維管理場景。
* JWT 無狀態認證 + BCrypt 密碼加密，雙重保障系統安全。
* AOP 自動日誌審計，無需手動錄入，實現全流程可追溯。
* 設備檢修 + 故障預警雙向管控，構建智能化設備運維體系。
